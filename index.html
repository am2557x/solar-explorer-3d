// SOLAR EXPLORER — H7006 WEB 3D APPLICATION

// Project Overview

Solar Explorer is an interactive Web 3D application built for the H7006 module. Four celestial bodies — Earth, Saturn, Mars and the Moon — are presented as fully interactive GLB 3D models rendered via Three.js inside a space-themed HUD interface. Users can select between planets, orbit and zoom in 3D space, toggle wireframe view, control lighting, swap surface colour variants, and trigger animated launch sequences — all within a responsive layout with real scientific telemetry data displayed per planet.

// 3D Modelling

All four 3D models are integrated via Three.js GLTFLoader in GLB format. The GLB format encapsulates geometry, PBR materials, and texture data in a single binary file, making it optimal for web delivery. Models are rendered under a multi-point lighting rig with ambient, directional, and rim lighting. Wireframe inspection mode exposes the underlying polygon topology of each celestial body. A surface colour variant system traverses mesh children and applies THREE.Color updates in real time, with luminance thresholding to preserve dark detail regions.

// Design Rationale

The visual language is inspired by real NASA mission control interfaces and sci-fi HUD aesthetics. A near-black void background with blue glow accents creates the illusion of deep space, while Orbitron's geometric letterforms reinforce the space-age theme. CSS custom properties maintain consistent theming across all components. The three-column grid layout — navigation panel, 3D viewport, telemetry panel — mirrors a real mission control workstation. HUD corner brackets, a scanning line animation, and a live UTC clock add authentic atmosphere. The layout collapses to single-column at 900px for mobile compatibility.

// Interactivity & JavaScript Features

Three.js is loaded via ES Modules from the unpkg CDN. The render loop uses requestAnimationFrame for 60fps rendering. OrbitControls provide full drag-to-rotate and scroll-to-zoom interaction. Model switching uses GLTFLoader on demand, with proper scene cleanup and geometry disposal. Wireframe toggle traverses all mesh children flipping material.wireframe. A launch sequence animation applies timed Y-axis rotation. A surface colour variant system updates THREE.Color per mesh with luminance-based dark-detail preservation. An HDR environment map is loaded via RGBELoader for physically accurate image-based lighting with ACES Filmic tonemapping. An ambient space audio track is toggled via the HTML5 Audio API. Content swapping between Explore and About pages requires no page reload.

// Accessibility Considerations

All interactive elements include ARIA labels and roles. Planet selection buttons support keyboard navigation via Tab and Enter keys. The scan line and starfield animations are marked aria-hidden to avoid screen reader noise. Colour contrast across all text elements meets WCAG 2.1 AA requirements. The prefers-reduced-motion CSS media query disables the scan line and star animations for users who require it. Semantic HTML5 elements (header, main, aside) ensure correct screen reader traversal order.

// Testing Strategy

Live testing was conducted across Chrome 120, Firefox 121, and Safari 17 on macOS and iOS. Three.js GLTFLoader performance was profiled using Chrome DevTools — planet model load times were under 600ms per asset. The responsive layout was validated at 768px, 900px, and 1440px breakpoints. The UTC clock and telemetry data update cycle were verified for accuracy. Peer testing feedback in Week 11 resulted in improved lighting controls and the addition of surface colour swatches.
